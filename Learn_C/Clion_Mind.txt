#include <XXX> 与 #include "XXX"
  使用尖括号 <> 表示：编译器应该从标准库的路径中查找该头文件；
  使用双引号""表示：编译器会在当前源代码文件的路径中查找该头文件（用于用户自定义或项目内部的头文件）

头文件 与 共享库（动态链接库）：头文件可以在编译时被不同C++源代码共享声明使用；共享库（动态链接库）可以在运行时被不同程序共享运行使用
  头文件：
    1、头文件包含函数、类、变量的声明，宏定义、类型别名等代码，但是它们不包含实际的实现代码
    2、.h、.hpp、.hxx 都是用于声明和共享代码接口的 C++ 头文件

  共享库（动态链接库）：
    1、共享库包含已编译的二进制代码，实现了各种函数和功能
    2、.so 文件是 Linux/Unix 系统上的共享库或动态链接库的文件后缀。在 Windows 上，对应的是 .dll 文件

  动态链接库（Dynamic Link Library，DLL [.dll 或 .so 文件]）： 包含已编译的二进制代码，在程序运行时会被加载到内存中。
    1、DLL文件独立于程序存在，多个程序可以共享一个DLL，减少了代码冗余和内存占用。

  静态链接库（Static Link Library，LIB [.lib 或 .a 文件]）：包含未编译的代码，在编译时会被完整地复制到每个使用它的程序中。
    1、LIB文件使得程序在运行时不需要额外加载库文件
